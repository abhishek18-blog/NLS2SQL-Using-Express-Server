====================================================================
DATABASE.TS - COMPLETE CODE BREAKDOWN
====================================================================
This file acts as the bridge between your Node.js server and your 
actual MySQL database. It handles connections, queries, and extracting 
the database structure.

--------------------------------------------------------------------
1. FUNCTIONS & METHODS USED
--------------------------------------------------------------------
- getDbUrl()
  What it is: A custom helper function.
  Why: It builds the secure MySQL connection string using variables hidden in your `.env` file (like user, password, port).

- mysql.createConnection()
  What it is: A method from the `mysql2/promise` library.
  Why: This physically reaches out over the network and logs into your MySQL database server.

- connection.query()
  What it is: A method attached to the connection object.
  Why: This is what actually fires the raw SQL string (e.g., "SHOW TABLES;") into the database and fetches the data back.

- connection.end()
  What it is: Connection closer method.
  Why: If you don't close the connection after you finish querying, your Node server will run out of memory and database slots.

- split() & pop()
  What it is: JavaScript string/array methods.
  Why: Used to slice up the `dbUrl` to extract just the specific database name (like "sakila") from a long URL.

- Object.values()
  What it is: A built in JavaScript method.
  Why: Extracts just the raw values from a confusing object (e.g., turning `{ Tables_in_sakila: "users" }` into just `["users"]`).

- map()
  What it is: Array formatting method.
  Why: Loops through the raw database columns array and transforms each one into a clean string format (identifying PKs and FKs).

- join(",")
  What it is: Array glue method.
  Why: Takes the array generated by map() and smashes it into a single clean string of comma-separated column names for the AI to read.

--------------------------------------------------------------------
2. VARIABLES
--------------------------------------------------------------------
- dbName: Optional input parameter to specify a specific database.
- addLog: Optional callback function to push logs up to the frontend UI.
- dbUrl: Stores the fully constructed MySQL connection string.
- connection: Stores the active, live session with the database.
- tables: An array containing the raw results of the "SHOW TABLES;" query.
- schemaStr: A string that starts empty and gradually grows to contain the massive map of your entire database schema.
- row: A temporary variable inside a `for...of` loop representing one table at a time.
- tableName: A clean, extracted string containing just the name of the current table.
- columns: An array of metadata describing every column inside the current table.
- columnDefs: A highly formatted, single string containing all columns for a table (e.g. "id :PK, email, name").
- col: A temporary variable inside the `.map()` function representing one specific column's data.

--------------------------------------------------------------------
3. KEYWORDS
--------------------------------------------------------------------
- export: Allows functions like getSchema and execute to be imported by server.ts and ai.ts.
- async / await: Crucial here, because talking to a database over a network is slow. The code must wait for the database to reply.
- try / catch: Used to catch database authentication errors or bad SQL syntax so the server doesn't crash.
- const / let: 'let' is specifically used for `schemaStr` because we constantly append new text to it in a loop.
- for...of: A loop keyword used to cycle through the array of `tables` one by one.
- as: A TypeScript keyword (Type Assertion). E.g., `as string` forces TypeScript to treat the extracted table name as a string to prevent type-checking errors.
- return: Exits the function and passes the final schema string (or data records) back to the file that requested it.
